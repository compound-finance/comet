# Partial Liquidation — Worked Example

Повний приклад від депонування до виконання часткової ліквідації з досягненням `targetHF = 1.05`.

---

## Параметри протоколу

| Параметр | COMP | ETH |
|---|---|---|
| `borrowCollateralFactor` (CF) | 0.75 | 0.80 |
| `liquidateCollateralFactor` (liquidateCF) | 0.80 | 0.85 |
| `liquidationFactor` (LP) | 0.90 | 0.90 |

```
targetHF    = 1.05
Base token  = USDC ($1.00)
```

---

## Фаза 1 — Депонування та позика

### Дії користувача

| Дія | Актив | Кількість | Ціна | Вартість |
|---|---|---|---|---|
| Депозит | COMP | 100 токенів | $20.00 | $2,000.00 |
| Депозит | ETH | 0.5 токена | $2,000.00 | $1,000.00 |
| Позика | USDC | — | — | $1,800.00 |

### Розрахунок початкового стану

**Вартість застави:**

| Актив | Raw USD | × CF | CF-weighted | × liquidateCF | liquidateCF-weighted |
|---|---|---|---|---|---|
| COMP | $2,000.00 | 0.75 | $1,500.00 | 0.80 | $1,600.00 |
| ETH | $1,000.00 | 0.80 | $800.00 | 0.85 | $850.00 |
| **Всього** | **$3,000.00** | | **$2,300.00** | | **$2,450.00** |

**Ключові метрики:**

```
Борг (debt)                   = $1,800.00 USDC

HF  = CF-weighted / debt      = $2,300 / $1,800     = 1.2778
LTV = debt / raw collateral   = $1,800 / $3,000     = 60.00%

Доступна кредитна ємність     = $2,300 − $1,800     = $500.00
Утилізація позики             = $1,800 / $2,300     = 78.26%

isLiquidatable?
  liquidateCF-weighted ($2,450) > debt ($1,800)  →  НІ ✓
```

---

## Фаза 2 — Падіння ціни COMP

Ціна COMP падає з **$20.00** до **$11.00** (−45%). Ціна ETH залишається $2,000.00.

### Перерахунок вартості застави

| Актив | Кількість | Нова ціна | Raw USD | CF-weighted | liquidateCF-weighted |
|---|---|---|---|---|---|
| COMP | 100 | $11.00 | $1,100.00 | $825.00 | $880.00 |
| ETH | 0.5 | $2,000.00 | $1,000.00 | $800.00 | $850.00 |
| **Всього** | | | **$2,100.00** | **$1,625.00** | **$1,730.00** |

### Ключові метрики після падіння ціни

```
Борг (debt)                   = $1,800.00 USDC (незмінний)

HF  = CF-weighted / debt      = $1,625 / $1,800     = 0.9028   ← нижче 1.0!
LTV = debt / raw collateral   = $1,800 / $2,100     = 85.71%

isLiquidatable?
  liquidateCF-weighted ($1,730) < debt ($1,800)  →  ТАК ✓

Дефіцит ліквідності           = $1,800 − $1,730     = $70.00
```

### Порівняння метрик: до і після падіння ціни

| Метрика | Початок | Після падіння | Зміна |
|---|---|---|---|
| Ціна COMP | $20.00 | $11.00 | −45% |
| Raw collateral | $3,000.00 | $2,100.00 | −$900 |
| CF-weighted | $2,300.00 | $1,625.00 | −$675 |
| liquidateCF-weighted | $2,450.00 | $1,730.00 | −$720 |
| **HF** | **1.2778** | **0.9028** | **−0.375** |
| **LTV** | **60.00%** | **85.71%** | **+25.71 pp** |
| isLiquidatable | НІ | **ТАК** | — |

## Фаза 3 — Виконання часткової ліквідації

### Перевірка умови валідності конфігурації

Перед розрахунком перевіряємо знаменник `LP × targetHF − CF > 0` для кожного активу:

```
COMP:  0.90 × 1.05 − 0.75 = 0.945 − 0.75  = 0.195  > 0  ✓
ETH:   0.90 × 1.05 − 0.80 = 0.945 − 0.80  = 0.145  > 0  ✓
```

Обидва активи можуть брати участь у частковій ліквідації.

---

### Стан на початку циклу

```
debt_remaining    = $1,800.00
totalCV_CF        = $1,625.00   (CF-weighted залишок усіх активів)
deltaValue        = $0.00       (накопичене погашення боргу)
```

---

### Ітерація 1: COMP (актив 0)

**Крок 1 — Розрахунок необхідного сейзу за формулою (RFC formula 13)**

```
Δ_COMP = (targetHF × debt_remaining − totalCV_CF) / (LP_COMP × targetHF − CF_COMP)

       = (1.05 × $1,800.00 − $1,625.00) / (0.90 × 1.05 − 0.75)

       = ($1,890.00 − $1,625.00) / (0.945 − 0.75)

       = $265.00 / 0.195

       = $1,358.97
```

**Крок 2 — Порівняння з наявним балансом**

```
Доступно COMP:  100 × $11.00 = $1,100.00
Потрібно:                      $1,358.97

$1,100.00 < $1,358.97  →  недостатньо  →  ПОВНЕ вилучення COMP
```

**Крок 3 — Повне вилучення COMP**

```
seizeAmount  = 100 COMP
rawUSD       = 100 × $11.00   = $1,100.00
seizedValue  = $1,100.00 × LP = $1,100.00 × 0.90 = $990.00   (погашення боргу)

deltaValue   = $0.00 + $990.00 = $990.00
```

**Крок 4 — Оновлення залишкових значень**

```
debt_remaining  = $1,800.00 − $990.00  = $810.00
totalCV_CF      = $1,625.00 − $825.00  = $800.00
                           (CF-weighted COMP = 100 × $11 × 0.75 = $825)
```

---

### Ітерація 2: ETH (актив 1)

**Крок 1 — Розрахунок необхідного сейзу**

```
Δ_ETH = (targetHF × debt_remaining − totalCV_CF) / (LP_ETH × targetHF − CF_ETH)

      = (1.05 × $810.00 − $800.00) / (0.90 × 1.05 − 0.80)

      = ($850.50 − $800.00) / (0.945 − 0.80)

      = $50.50 / 0.145

      = $348.28
```

**Крок 2 — Порівняння з наявним балансом**

```
Доступно ETH:   0.5 × $2,000.00 = $1,000.00
Потрібно:                          $348.28

$348.28 ≤ $1,000.00  →  достатньо  →  ЧАСТКОВЕ вилучення ETH ✓
```

**Крок 3 — Часткове вилучення ETH**

```
rawCollateralUSD = $348.28

seizeAmount  = $348.28 / $2,000.00      = 0.17414 ETH
seizedValue  = $348.28 × LP             = $348.28 × 0.90 = $313.45   (погашення боргу)

deltaValue   = $990.00 + $313.45 = $1,303.45
```

**Крок 4 — targetHF досягнуто → BREAK**

```
currentHF = targetHF = 1.05  →  вихід з циклу
```

---

### Фінальний розрахунок нового балансу

```
Загальне погашення боргу (deltaValue) = $1,303.45 USDC

Новий борг = $1,800.00 − $1,303.45 = $496.55 USDC
```

---

## Фаза 4 — Стан після часткової ліквідації

### Залишки застави

| Актив | До ліквідації | Вилучено | Залишок | Raw USD залишку |
|---|---|---|---|---|
| COMP | 100.00000 | 100.00000 | **0.00000** | $0.00 |
| ETH | 0.50000 | 0.17414 | **0.32586** | $651.72 |

### Перевірка досягнення targetHF

```
Залишок ETH raw USD:
  0.32586 ETH × $2,000.00 = $651.72

CF-weighted залишок:
  $651.72 × 0.80 = $521.38

Новий борг:
  $496.55 USDC

HF_post = CF-weighted / debt = $521.38 / $496.55 = 1.0500  ✓  (= targetHF)
```

Перевірка алгебраїчно:
```
New TCV = TCV_after_COMP − Δ_ETH × CF_ETH
        = $800.00 − $348.28 × 0.80
        = $800.00 − $278.62
        = $521.38

New debt = debt_remaining − seizedValue_ETH
         = $810.00 − $313.45
         = $496.55

HF = $521.38 / $496.55 = 1.0500  ✓
```

### Ключові метрики після ліквідації

| Метрика | До ліквідації (з падінням) | Після ліквідації | Зміна |
|---|---|---|---|
| Борг (USDC) | $1,800.00 | **$496.55** | −$1,303.45 |
| Raw collateral | $2,100.00 | **$651.72** | −$1,448.28 |
| CF-weighted | $1,625.00 | **$521.38** | −$1,103.62 |
| liquidateCF-weighted | $1,730.00 | **$554.27** | −$1,175.73 |
| **HF** | **0.9028** | **1.0500** | **+0.147** |
| **LTV** | **85.71%** | **76.19%** | −9.52 pp |
| isLiquidatable | ТАК | **НІ** | — |

```
liquidateCF-weighted ($554.27) > борг ($496.55)  →  НЕ ліквідований ✓
```

## Покрокова схема алгоритму

```
absorb(account)
│
├─ isLiquidatable?
│    liquidateCF-weighted ($1,730) < debt ($1,800)  →  ТАК
│
├─ Pre-calc:
│    totalCV_CF  = $1,625   (CF-weighted)
│    debt        = $1,800
│
├─ Актив 0: COMP
│    Δ = (1.05×1800 − 1625) / (0.9×1.05 − 0.75) = 265/0.195 = $1,358.97
│    available  = $1,100  <  $1,358.97
│    → ПОВНЕ вилучення 100 COMP
│    seizedValue = $1,100 × 0.9 = $990  (борг погашено)
│    debt_remaining  = $810
│    totalCV_CF      = $800
│    currentHF < targetHF  →  continue
│
├─ Актив 1: ETH
│    Δ = (1.05×810 − 800) / (0.9×1.05 − 0.80) = 50.5/0.145 = $348.28
│    available  = $1,000  ≥  $348.28
│    → ЧАСТКОВЕ вилучення 0.17414 ETH
│    seizedValue = $348.28 × 0.9 = $313.45  (борг погашено)
│    currentHF = targetHF = 1.05  →  BREAK
│
└─ Фінал:
     Новий борг   = $1,800 − ($990 + $313.45) = $496.55 USDC
     Залишок ETH  = 0.32586 токенів
     HF           = $521.38 / $496.55 = 1.0500  ✓
```

## Примітки

**Чому COMP вилучено повністю, а ETH частково?**
Падіння ціни COMP на 45% зробило його внесок у CF-weighted collateral ($825) недостатнім для досягнення targetHF самостійно. Алгоритм вилучає COMP повністю і переходить до ETH, де вистачає лише $348.28 з $1,000 доступних.

**Що таке `seizedValue` у кожному кроці?**
`seizedValue = rawUSD × LP` — це сума боргу, що списується за вилучення цього забезпечення. LP = 0.9 означає, що за $1,000 вилученого COMP борг зменшується лише на $900 — різниця $100 залишається протоколу як ліквідаційна премія для ліквідаторів через механізм `buyCollateral`.

**Чому `isLiquidatable = false` після часткової ліквідації?**
Часткова ліквідація доводить акаунт до `HF = targetHF = 1.05`. Оскільки `liquidateCF > borrowCF`, після досягнення `HF_borrowCF = 1.05` завжди виконується:
```
HF_liquidateCF ≈ HF_borrowCF × (liquidateCF / borrowCF) = 1.05 × 1.0625 ≈ 1.116 > 1.0
```
Тобто liquidateCF-weighted collateral перевищує борг → акаунт здоровий за стандартом ліквідації.
